# [BUG-002] Reader UI scrolling jumps to beginning
- **Date reported**: YYYY-MM-DD
- **Reported by**: Agent
- **Severity**: High
- **Status**: Resolved
- **Component**: Flutter UI
- **Description**: In the Reader UI, the UI automatically refreshes to add a new page (from the background OCR/AI pipeline), which results in a restart of the book view (it jumps to the beginning). Additionally, the SQLite persistent page tracking does not correctly remember the last page.
- **Expected behavior**: The UI should seamlessly append the new page without losing the user's scroll position, and reopening the book should start on the last read page.
- **Root cause analysis**: 
  1. The `_refreshTimer` in `reader_screen.dart` calls `ref.invalidate(readerViewProvider)`, which destroys the provider and removes the list from the UI during the loading phase. This causes the `ListView` to unmount and reset its scroll offset.
  2. The `_scheduleRestore` method in `reader_screen.dart` sets a target index of 0 if the target page has not loaded yet (`index == -1`). It prematurely restores the view to index 0, effectively jumping to the top and saving page 0 into the SQLite DB, breaking persistence.
- **Fix applied**:
  - Files changed: `lib/providers/reader_provider.dart`, `lib/ui/screens/reader/reader_screen.dart`
  - Approach: Introduced a `readerRefreshProvider` counter to refresh the provider without invalidating/destroying its previous state. Modified `_restoreToPage` to abort if `index == -1`, waiting instead for the page to be available.
