# [BUG-003] Tesseract data downloaded despite being bundled
- **Date reported**: YYYY-MM-DD
- **Reported by**: Agent
- **Severity**: Medium
- **Status**: Resolved
- **Component**: Kotlin plugin
- **Description**: Despite bundling `ben` and `eng` Tesseract traineddata, the app attempts to download the data again when a combination code like `ben+eng` is passed.
- **Expected behavior**: The plugin should recognize that `ben` and `eng` are installed individually and skip the download.
- **Root cause analysis**: The `ensureTessData` function in `LeafPlugin.kt` checked for the exact filename representation of the `tessCode` (e.g., `ben+eng.traineddata`). Tesseract parses `ben+eng` by searching for both `ben.traineddata` and `eng.traineddata` individually.
- **Fix applied**:
  - Files changed: `android/app/src/main/kotlin/com/spark/leaf/LeafPlugin.kt`
  - Approach: Split the `tessCode` by `+` and ensured the existence (or initiated download) for each individual locale file before proceeding.
