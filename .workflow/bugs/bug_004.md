# Bug 004: Cannot Create Books After Changing Settings Directory

**Date discovered:** 2026-04-02
**Status:** Resolved

## Description
When the user changed the directory in settings to an external directory via FilePicker (on Android), they could no longer create new books. The `createBookFolder` function threw a `FileSystemException: Permission denied` silently, breaking the book creation flow. Additionally, the "Change Saved Directory" button in Settings had an inconsistent styling compared to other primary buttons.

## Root Cause
The `FilePicker.platform.getDirectoryPath()` method on Android relies on Scoped Storage, and standard `dart:io` operations require the `MANAGE_EXTERNAL_STORAGE` permission to write to arbitrary external paths chosen by the user. Since the app didn't declare or request this permission, any `File` or `Directory` creation inside the newly picked path failed.

## Proposed Fix
- Added the `permission_handler` package dependency.
- Declared `READ_EXTERNAL_STORAGE`, `WRITE_EXTERNAL_STORAGE` and `MANAGE_EXTERNAL_STORAGE` in `AndroidManifest.xml`.
- Added runtime permission requests in `SettingsScreen._changeLibraryDirectory` prior to picking a folder.
- Updated the "Change Saved Directory" button style to `ElevatedButton.icon` to be consistent with UI.
