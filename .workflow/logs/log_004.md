## [LOG-004] Tesseract Engine Configuration Update
- **Date**: 2026-04-02
- **Agent**: Antigravity
- **Session type**: Engine Logic Refactoring
- **What was done**:
  - Investigated the usage of `ben+eng` combined layout logic within the application.
  - Modified `LeafPlugin.kt` to completely eliminate the combined `ben+eng` Tesseract initialization.
  - Tesseract is now configured to load the `ben` and `eng` models purely and independently.
- **Why**: Requested by the user because mixing `ben+eng` results in poor Bengali OCR processing. Tesseract should use pure `ben` for Bengali reading without introducing English bias where it's not strictly necessary. 
- **Files changed**:
  - `android/app/src/main/kotlin/com/spark/leaf/LeafPlugin.kt`
- **Outcome**: The OCR engine now natively spins up clean, separate instances for `ben` and `eng` directly during initialization. Bengali documents will hit the pure `ben` engine.
